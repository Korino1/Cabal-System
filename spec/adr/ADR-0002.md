# Фазовая структура, гейты и правило кодирования

## Контекст
Требуется повысить управляемость логической схемы без усложнения: единый формат фаз, прозрачные критерии входа/выхода и снижение контекстной нагрузки на исполнителей.

## Решение
- Ввести каноническую машиночитаемую структуру фазы (`spec/docs/PHASE_SCHEMA.md`).
- Ввести гейт‑чеклист входа/выхода фаз (`spec/docs/PHASE_GATE.md`).
- Явно закрепить правило: кодирование начинается только после завершения GA‑5.
- Закрепить протокол изменений `LOGIC_PROTOCOL.md` через ADR.
- Расширить фазовые INDEX разделами Inputs/Outputs/Evidence/Dependency Matrix.
- Добавить триггер обновления DIGEST по длительности активного WORKLOG.

## Альтернативы
- Оставить правила только в `LOGIC_PROTOCOL.md` без фазовых гейтов.
- Использовать исключительно свободную форму без фиксированной структуры фаз.

## Последствия
- Повышается предсказуемость и проверяемость фаз.
- Появляются дополнительные обязательные артефакты (PHASE_SCHEMA/PHASE_GATE).
- Требуется дисциплина обновления ADR при изменениях протокола.

## Связанные артефакты
- Документация: `spec/docs/PHASE_SCHEMA.md`, `spec/docs/PHASE_GATE.md`
- Протокол: `.memory/LOGIC_PROTOCOL.md`
